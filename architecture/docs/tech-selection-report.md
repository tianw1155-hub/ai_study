# 技术选型报告

## 1. 选型原则

基于弗雷德里克·布鲁克斯《人月神话》经验：
- **概念完整性优先**: MVP 阶段保持技术栈简洁统一
- **没有银弹**: 没有完美方案，只有最适合当前阶段的方案
- **第二系统效应警惕**: 避免过度设计，MVP 只做必需的技术决策

基于马丁·福勒《Clean Architecture》经验：
- **依赖倒置**: 核心业务逻辑不依赖具体技术实现
- **分层清晰**: 每层有明确职责，边界清晰

## 2. 技术选型矩阵

### 2.1 计算平台

| 候选 | 成本 | 运维复杂度 | 扩缩能力 | 冷启动 | 决策 |
|------|------|------------|----------|--------|------|
| 腾讯云 SCF | 低 (按调用) | 极低 | 自动 | 1-3s | **选用** |
| 腾讯云 CVM | 中 | 高 | 需手动 | 无 | 备选 |
| 腾讯云 TKE | 高 | 高 | 自动 | 10-30s | 剔除 |

**决策理由**: MVP 需要快速交付，SCF 的免运维特性可将运维工作量降至 0。

### 2.2 数据库

| 候选 | schema 灵活性 | 查询能力 | 运维复杂度 | 成本 | 决策 |
|------|---------------|----------|------------|------|------|
| MongoDB | 高 | 强 | 低 | 低 | **选用** |
| PostgreSQL | 中 | 强 | 中 | 中 | 备选 |
| MySQL | 低 | 中 | 中 | 低 | 剔除 |
| Redis | N/A | K/V | 低 | 中 | 缓存用 |

**决策理由**: 职业规划数据结构调整频繁，MongoDB 的灵活 schema 可减少后期迁移成本。

### 2.3 缓存

| 候选 | 集成难度 | 腾讯云支持 | Serverless | 成本 | 决策 |
|------|----------|------------|------------|------|------|
| 腾讯云 Redis | 原生 | 是 | 是 | 中 | **选用** |
| 内存缓存 | 简单 | 否 | 否 | 低 | 备选 |
| Memcached | 简单 | 否 | 否 | 低 | 剔除 |

**决策理由**: 腾讯云 Redis 与 SCF 集成简单，支持 Serverless 按需计费。

### 2.4 前端框架

| 候选 | SSR 支持 | SEO 友好 | 开发效率 | 学习成本 | 决策 |
|------|----------|----------|----------|----------|------|
| Next.js | 原生 | 是 | 高 | 中 | **选用** |
| Nuxt.js | 原生 | 是 | 高 | 中 | 备选 |
| 纯 Vue/React | 需配置 | 否 | 中 | 低 | 备选 |

**决策理由**: 职业平台需要 SEO 支持，Next.js 的 SSR 能力是刚需。

### 2.5 日志与监控

| 类型 | 候选 | 成本 | 集成难度 | 决策 |
|------|------|------|----------|------|
| 日志 | 腾讯云 CLS | 低 | 原生 | **选用** |
| 错误追踪 | Sentry | 免费额度 | 简单 | **选用** |
| APM | - | - | - | 暂不引入 |

**决策理由**: MVP 阶段只需要基础可观测性，避免过度工程化。

## 3. 成本估算 (MVP)

### 3.1 月度成本预估

| 服务 | 规格 | 单价 | 用量 | 月费 |
|------|------|------|------|------|
| SCF | 128MB, 100ms | 0.0002元/调用 | 100,000次 | ¥20 |
| API Gateway | 百万次 | ¥0.8/百万 | 1M | ¥0.8 |
| MongoDB Atlas | Serverless | ¥0.5/百万读 | 1M读 | ¥0.5 |
| Redis | 256MB | ¥0.3/GB/时 | 24h | ¥1.8 |
| COS | 标准存储 | ¥0.118/GB | 10GB | ¥1.18 |
| CLS | 按量计费 | ¥0.35/GB | 1GB | ¥0.35 |
| CDN | 按量计费 | ¥0.21/GB | 50GB | ¥10.5 |
| **总计** | | | | **≈¥35** |

### 3.2 成本优化策略

1. **冷数据降级**: 历史认证数据转 COS 归档
2. **缓存策略**: 热数据全量 Redis 缓存
3. **按需扩缩**: SCF 自动扩缩容，非高峰期零成本

## 4. 技术依赖图

```
用户请求
   ↓
腾讯云 CDN (静态资源) / API Gateway (动态请求)
   ↓
SCF 函数 (无状态)
   ↓
├── Redis (热数据缓存)
├── MongoDB (持久化数据)
└── COS (文件存储)
   ↓
CLS + Sentry (可观测性)
```

## 5. 供应商锁定风险评估

| 技术 | 锁定程度 | 替代方案 | 迁移成本 |
|------|----------|----------|----------|
| 腾讯云 SCF | 中 | 阿里云 FC / AWS Lambda | 中 |
| MongoDB Atlas | 中 | 自建 MongoDB / TencentDB | 中 |
| 腾讯云 Redis | 低 | 自建 / 其他云 | 低 |
| COS | 中 | AWS S3 / 阿里云 OSS | 中 |

**风险缓解**: 
- 使用标准 S3 API 兼容的 SDK 访问 COS
- MongoDB 使用官方驱动，非定制 API
