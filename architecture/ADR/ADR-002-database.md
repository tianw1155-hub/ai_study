# ADR-002: 数据库选型

**状态**: 已接受  
**日期**: 2026-03-22  
**决策者**: 架构团队

---

## 背景/问题陈述

MVP 需要存储以下核心数据：
- 用户信息 (手机号、昵称、技能档案)
- 职业路径 (技能要求、薪资范围)
- 技能图谱 (树形结构)
- 认证记录 (证书信息)

数据特点：
- Schema 变化频繁 (职业规划领域需求迭代快)
- 技能图谱是树形结构
- 需要灵活的查询能力

---

## 候选方案

### 方案 A: MongoDB

| 维度 | 评估 |
|------|------|
| Schema 灵活性 | **高** - 无固定 schema |
| 树形查询 | 支持 $graphLookup |
| 查询能力 | 强 (聚合管道) |
| 运维复杂度 | **低** (Atlas Serverless) |
| 成本 | 按使用计费 |
| 迁移成本 | 低 |

### 方案 B: PostgreSQL

| 维度 | 评估 |
|------|------|
| Schema 灵活性 | **中** - 需要 migrations |
| 树形查询 | 支持 LTREE 扩展 |
| 查询能力 | 强 |
| 运维复杂度 | 中等 |
| 成本 | 固定实例费 |
| 迁移成本 | 中 |

### 方案 C: MySQL

| 维度 | 评估 |
|------|------|
| Schema 灵活性 | **低** - 需要 ALTER TABLE |
| 树形查询 | 需递归 CTE 或邻接表 |
| 查询能力 | 中等 |
| 运维复杂度 | 中等 |
| 成本 | 低 |
| 迁移成本 | 低 |

---

## 决策

**选择方案 A: MongoDB Atlas Serverless**

### 决策理由

1. **Schema 灵活性**: 职业规划需求变化快，MongoDB 的灵活 schema 可减少迁移成本
2. **树形结构原生支持**: `$graphLookup` 可直接查询技能图谱
3. **运维最小化**: Atlas Serverless 免运维，自动扩缩
4. **MVP 友好**: 按使用计费，与流量成正比

### 限制条件

- 需要合理设计索引避免全表扫描
- 事务使用场景有限 (单个 document)
- 需要注意数据一致性问题

---

## 后果

### 正面

- 开发效率高，快速迭代
- Schema 变更无需 migration
- 按使用计费，成本可控
- 腾讯云原生集成

### 负面

- 事务能力弱 (单 document 级别)
- 关联查询不如 SQL
- 需要合理设计索引
