## 二、回归问题检查

### ⚠️ P2（轻微）：E-8 场景处理未在产物交付中完整定义

**问题位置：** PRD-产物交付.md §4.5 E-8 场景

**问题描述：** E-8 作为"部署进行中被回退"的异常场景，在异常处理表中仅描述了处理策略（"Vercel 调用中止 API / Render 提供提示"），但未定义：
1. Vercel 中止后应执行什么后续步骤
2. "强制中止"后的状态机走向（是等待 cancelled 还是直接标记 aborted 后继续回退流程）
3. E-8 在 rollback_logs 中的 step 编号

**影响评估：** 低 — E-8 属于边界场景，实现时可按策略描述自行设计；不影响核心回退功能正确性。

**建议：** 在 E-8 处理方式列中补充"强制中止后：直接进入 Step 3 重新部署，status 标记为 aborted"等后续动作描述。
