## 三、新发现的问题（按严重程度）

### 🔴 P0 - 必须修复

#### P0-1：M-3 延迟验收标准歧义
**问题：** 验收标准表格写"端到端 P95 延迟 < 1s"，但定义表格写总延迟 < 1.2s。测试用例无法确定用哪个阈值。

**建议修复：**
```
验收标准表格"指标"列修改为：
"端到端 P95 总延迟 < 1.2s（P95 网络延迟 < 1s + P95 渲染延迟 < 200ms）"
```

#### P0-2：M-4"站内通知"定义不清
**问题：** M-4 描述"站内通知正确送达"，但：
- 站内信存储在哪里？（数据库/WebSocket？）
- "WebSocket 消息计时"是测试 WebSocket 推送还是测试数据库记录？
- 如果 WebSocket 推送成功但数据库写入失败，算"送达"吗？

**建议修复：** 明确站内通知的存储和查询机制，或改为"WebSocket 通知推送成功"。

---

### 🟡 P1 - 应当补充

#### P1-1：敏感词检测测试集规模不足
**问题：** S-2 说"需准备测试集：正向≥100条，负向≥50条，含拼音/特殊字符边界"，但：
- 没有说明拼音/特殊字符边界用例的数量
- 没有说明"误拦截"和"漏拦截"的判定标准

**建议补充：**
```
敏感词检测测试集：
- 正向测试集：100条正常需求（覆盖短文本/长文本/特殊业务术语）
- 负向测试集：50条包含敏感词的需求
- 边界测试集：
  * 拼音拆分：10条（如"f u c k"）
  * 特殊字符：10条（如"f*ck"、"f【】ck"）
  * 谐音：10条
  * 英文缩写：10条
- 验收标准：
  * 误拦截率 = 被误判的正向用例 / 正向总数 < 1%
  * 漏拦截率 = 未检出的负向用例 / 负向总数 < 0.1%
```

#### P1-2：敏感词检测时序不明
**问题：** M1.1 表格写"敏感词检测：提交时触发"，但没有说明：
- 检测是同步还是异步？
- 检测失败是否阻塞提交？
- 检测超时时间？

**建议补充：** 明确检测时序和失败处理逻辑。

#### P1-3：API 响应格式缺失
**问题：** 第6节列出了 API 签名，但没有提供响应格式和错误码定义：
- `/api/requirements/submit` - 成功响应结构？错误码？
- `/api/documents/parse` - 响应格式已在流程中描述，但缺少错误码
- `/api/sensitive/check` - 响应格式完全缺失

**建议补充：** 各 API 的成功/失败响应 JSON 结构。

### 🟠 P2 - 建议优化

#### P2-1：文件上传大小超限处理不明
**问题：** M1.2 规定单文件最大 10MB，但没说超过时的 UI 表现。

**建议：** 补充"超过10MB时，前端拦截并提示'文件大小超出限制（最大10MB）'"。

#### P2-2：WebSocket 认证失败处理缺失
**问题：** 说明 JWT 认证方式，但没说认证失败（如 token 过期）的处理逻辑。

**建议：** 补充"认证失败时，服务端关闭连接并返回错误码，前端跳转登录页"。

#### P2-3：动态流折叠后交互细节缺失
**问题：** S-3 要求"可查看最近50条动态"，但：
- 折叠后如何展开？"展开更多"按钮在点击时是一次性展开所有还是分页？
- 50条是前端存储还是后端提供？

**建议：** 明确展开逻辑和数据来源。
