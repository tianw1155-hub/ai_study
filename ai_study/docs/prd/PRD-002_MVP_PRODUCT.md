# PRD-002：云智开发平台 MVP 产品需求

> 文档状态：草稿 | 版本：v0.1 | 创建日期：2026-03-26

---

## 1. 产品愿景

**一句话定位：**
一个让用户用自然语言提交需求、AI agent 自动完成开发、测试、交付全流程的智能开发平台。

**核心价值：**
- 零开发基础也能提交需求
- AI 全自动处理，无需人工盯进度
- 随时查看交付状态和产物

---

## 2. 用户与场景

### 目标用户
外部用户（开发者、产品经理、创业者），通过 GitHub OAuth 注册登录。

### 核心场景
1. 用户提交一个需求（自然语言 or 上传文档）
2. AI 自动完成：PRD 生成 → 开发 → 测试
3. 用户在网页上查看进度、获取代码

---

## 3. 技术架构

| 层级 | 技术选型 |
|------|----------|
| 前端 | React / Next.js |
| 后端 | Go + Gin |
| Agent 层 | Python + LangGraph |
| LLM | 用户自选（GPT-4 / Claude / 国产模型） |
| 数据库 | PostgreSQL |
| 部署 | 腾讯云 Docker |

### 第三方集成
- **GitHub OAuth**：用户登录 + 代码仓库访问
- **LLM API**：由用户自行配置 API Key

---

## 4. 功能范围（MVP）

### 界面一：首页

**目标：** 用户登录后的落地页，核心操作 + 实时动态的入口。

包含：
- **M1: 需求提交入口**（+ 新增需求 / + 上传需求文档 / + 优化建议）
- **M2: 实时动态流**（展示所有 agent 的工具调用记录）
- **M3: 结果通知**（任务完成/失败时在首页直接弹出提示）

### 界面二：任务看板

**目标：** 可视化查看自己需求的任务执行状态。

包含：
- **M1: Kanban 看板**（5列：待处理 / 进行中 / 等待中 / 已完成 / 失败）
- **M2: 任务状态流转**（状态机控制，不允许跨状态跳）
- **M3: 任务详情**（执行日志、错误信息、关联产物）

### 界面三：产物交付

**目标：** 查看历史交付物，获取代码。

包含：
- **M1: PRD 版本管理**（所有评审通过版本可查）
- **M2: 代码交付**（GitHub 跳转链接）
- **M3: 回退机制**（PRD + 代码同步回退）

---

## 5. 用户隔离策略

| 维度 | 策略 |
|------|------|
| 数据隔离 | 用户只能看到自己提交且已审批的需求相关的任务和产物 |
| 仓库权限 | 仅支持用户个人仓库，不支持组织仓库 |
| Fork 限制 | 禁止 Fork ai_study 仓库（配置层面拦截） |
| API Key | 用户自行配置，自行承担费用 |

---

## 6. 通知与反馈机制

### 首页通知（MVP 唯一通知渠道）

| 事件 | 提示方式 |
|------|----------|
| 需求提交成功 | 首页顶部横幅："需求已提交，评审中..." |
| PRD 评审完成 | 首页顶部横幅："PRD 已通过评审，开发中..." |
| 开发完成 | 首页顶部横幅 + 动态流高亮："代码已生成，待测试" |
| 测试完成 | 首页顶部横幅："测试通过，交付完成" |
| 任务失败 | 首页顶部红色横幅 + 动态流红色标记 |

> **说明：** MVP 不做飞书/TG 渠道通知，所有反馈集中在首页。

---

## 7. 用户操作流程

```
访问网站
    ↓
GitHub OAuth 登录
    ↓
进入首页（落地页）
    ↓
提交需求（自然语言 / 上传文档）
    ↓
系统自动处理：PM → Dev → Test
    ↓
首页实时通知 + 动态流更新
    ↓
任务看板查看详细进度
    ↓
产物交付查看 PRD + 代码
```

## 9. MoSCoW 定级（MVP）

| 级别 | 功能 |
|------|------|
| **Must** | GitHub OAuth 登录、需求提交（自然语言）、首页动态流、任务看板（Kanban）、PRD 版本展示、代码 GitHub 跳转 |
| **Should** | 首页通知横幅、任务详情、文档上传提交、PRD + 代码同步回退 |
| **Could** | 优化建议提交、执行日志展开、错误堆栈详情 |
| **Won't** | 飞书/TG 通知、房间动画、多选批量操作、代码内嵌展示 |
