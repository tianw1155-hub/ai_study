## 一、审核结论

v0.3 相比 v0.2 有实质性改进，P0 修复项大部分已落地。但综合两份评审，发现 **4 个 P0 阻塞性问题** 需在 v0.4 中修复，另有 2 个 P1 跨模块一致性问题。

---

## 二、P0 阻塞性问题（必须修复）

### P0-1：WebSocket 认证方式三处文档均未彻底统一 🔴

**发现来源：** 开发评审（DEV）+ 测试评审（TEST），两者均独立指出同一问题

**问题描述：**
- 首页正文 M2.3 已改用 `Sec-WebSocket-Protocol` 头
- 但第 6 章 API 签名表仍写 `wss://api.devpilot.com/ws?token={jwt}`
- 任务看板第 4 章仍写 `wss://api.devpilot.com/ws?token={jwt_token}`
- 产物交付未明确定义，但复用了 SYSTEM PRD 的 URL query params 方案

**审核意见：**
这是 v0.3 修订最不彻底的一项。正文的 JWT 安全修复（P0-3）写清楚了，但 API 签名表和任务看板 WebSocket 节没有同步修订。**建议修复方式：**
1. 首页 API 签名表：`wss://api.devpilot.com/ws`（认证通过 `Sec-WebSocket-Protocol` 头）
2. 任务看板 WebSocket 节：同上
3. 产物交付：明确使用同一 WebSocket 连接，补充说明

---

### P0-2：E-1 异常处理 rollback_logs step 编号错误 🔴

**发现来源：** 开发评审（DEV）

**问题描述：**
产物交付 4.2 流程图已调整步骤顺序（Step1=Git revert，Step2=数据库），但 4.5 节 E-1 仍写"step=2 标记 failed"。这是文档内部不一致，开发者若按 E-1 实现会写出错误的补偿逻辑。

**审核意见：**
E-1 应标记 `step=1`（Git revert 失败）。修复极简单，但影响极大，必须在 v0.4 中更正。

---

### P0-3：重试间隔无量化验收标准 🔴

**发现来源：** 测试评审（TEST）+ 开发评审（DEV）

**问题描述：**
任务看板 M2.5 定义了指数退避（10s → 30s → 90s），但验收标准里只有 M-6 验证 retryCount +1，**没有验证时间间隔本身**。测试无法判断退避实现是否正确。

**审核意见：**
建议在 Should 验收标准中增加 S-x：
> "重试触发时间符合指数退避（允许 ±10% 误差），测量方式：记录每次重试的 actual_delay，验证 10s ≤ actual_delay[1] ≤ 11s，27s ≤ actual_delay[2] ≤ 33s，81s ≤ actual_delay[3] ≤ 99s"

---

### P0-4："三端一致"定义不明确，无法编写测试断言 🔴

**发现来源：** 测试评审（TEST）

**问题描述：**
产物交付 M-5 写"回退后代码/PRD/预览三端一致"，但未定义三端具体指什么。测试无法写出精确断言。

**审核意见：**
建议明确定义：
> - **代码端：** GitHub revert commit SHA 与目标版本 commit SHA 一致
> - **PRD 端：** 数据库 `current_version_id` 指向目标版本
> - **预览端：** preview_url 指向的 deployment_id 对应目标版本 commit

三个值两两比对一致，才算"三端一致"。

## 三、P1 跨模块一致性问题（强烈建议修复）

### P1-1：HiL #2 机制跨模块未定义 🟡

**发现来源：** 开发评审（DEV）

**问题描述：**
产物交付和任务看板均提及"HiL #2 确认部署"，但 SYSTEM PRD 中无 HiL #2 的接口定义。这是一个依赖 SYSTEM PRD 的外部组件，必须明确。

**审核意见：**
建议在 SYSTEM PRD 中补充 HiL #2 的接口签名，或在相关 PRD 中注明"HiL #2 为系统外部依赖，其接口定义不在本 PRD 范围内，由系统架构文档约束"。

---

### P1-2：task:state_changed 事件首页未定义 🟡

**发现来源：** 开发评审（DEV）

**问题描述：**
任务看板依赖 `task:state_changed` 事件驱动看板更新（from_state/to_state），但首页 WebSocket 事件表中未定义此事件。前端需要两套事件定义，需要对齐。

**审核意见：**
首页 M2.3 WebSocket 事件表应补充 `task:state_changed` 事件，与任务看板保持一致。

## 四、两份评审的差异点分析

| 问题 | DEV 判断 | TEST 判断 | 审核结论 |
