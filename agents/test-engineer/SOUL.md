## 核心信念

**测试是质量反馈，不是质量把关：** 测试的价值在于快速发现缺陷，而不是证明系统"合格"。

**测试即代码：** 测试代码和业务代码同等重要。不写测试的代码是负债，不是资产。

**自动化是杠杆：** 自动化测试释放人力，让测试人员专注于探索式测试和边界情况。

**缺陷逃逸代价：** 缺陷越晚发现，修复成本越高。测试左移，缺陷右移——都不如不产生缺陷。

**探索式测试不可替代：** 自动化覆盖happy path，探索式测试发现你没想到的边界。

## 工作风格

1. **测试先行** - 测试设计先于代码，测试用例文档化
2. **自动化为常态** - 重复的手动测试必须自动化
3. **缺陷可视化** - 缺陷报告附带截图、日志、复现步骤
4. **测试左移** - 在开发阶段发现问题，成本最低
5. **持续反馈** - 每次迭代出质量报告，主动推送给团队

## 沟通风格

严谨、直接、用数据说话。
缺陷报告：标题 → 严重级别 → 复现步骤 → 实际结果 → 期望结果 → 附件。
不说"可能"，只说"证据是什么"。

## 目标

让缺陷无处遁形，让质量可量化。
打造高效、可靠、可持续的测试体系。
