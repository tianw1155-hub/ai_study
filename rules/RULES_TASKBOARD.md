# RULES_TASKBOARD.md - 任务看板模块规则 v1.0

> 文档状态：草稿 | 创建日期：2026-03-26

---

## 模块结构

```
TaskBoard
├── M1: Kanban 看板
├── M2: 任务状态流转
└── M3: 任务详情
```

---

## M1: Kanban 看板

### 职责
以 Kanban 形式展示 4 个 agent 的实时任务状态，供用户一目了然掌握项目整体进度。

### 看板列定义

| 列名 | 含义 | 包含状态 |
|------|------|----------|
| **待处理** | 任务已创建，尚未开始 | `pending` |
| **进行中** | 任务正在执行 | `in_progress` |
| **等待中** | 等待外部依赖或人工确认 | `waiting` |
| **已完成** | 任务执行成功 | `completed` |
| **失败** | 任务执行失败 | `failed` |

### 任务卡片信息

每个卡片展示：
- 任务名称
- 所属 agent（PM / Dev / Test / Ops）
- 当前状态（彩色标签）
- 创建时间
- 预计/实际耗时（如有）

### 限制

| 编号 | 限制内容 |
|------|----------|
| C1.1 | 卡片按创建时间倒序排列 |
| C1.2 | 单列卡片数量不设上限 |
| C1.3 | 不支持用户手动拖拽改变任务状态（状态由系统自动更新） |
| C1.4 | 失败卡片显示红色边框，进行中卡片显示蓝色边框 |
| C1.5 | 不支持多选批量操作 |
| C1.6 | 不做房间动画，仅状态文字/颜色变化 |

### 验收标准

- [ ] 4个 agent 的任务正确归类到对应列
- [ ] 任务状态变更后看板实时刷新（≤5秒）
- [ ] 失败任务有明确视觉标记
- [ ] 点击卡片可展开任务详情

## M2: 任务状态流转

### 职责
定义任务在各状态之间的流转规则，确保状态机清晰可追溯。

### 状态机

```
     创建
       ↓
    待处理 ──→ 进行中
       ↑           ↓
       ←── 等待中 ←┘
                   ↓
               完成 / 失败
```

### 流转规则

| 当前状态 | 可流转至 | 触发条件 |
|----------|----------|----------|
| `pending` | `in_progress` | Agent 开始执行 |
| `pending` | `waiting` | 等待外部依赖 |
| `in_progress` | `waiting` | 外部依赖阻塞 |
| `in_progress` | `completed` | 任务成功完成 |
| `in_progress` | `failed` | 执行报错或超时 |
| `waiting` | `in_progress` | 依赖解除 |
| `waiting` | `failed` | 依赖永久不可用 |
| `failed` | `pending` | 重新发起任务 |

### 限制

| 编号 | 限制内容 |
|------|----------|
| C2.1 | 不允许跨状态跳转（如 pending → completed） |
| C2.2 | 每次流转记录时间戳 |
| C2.3 | 失败任务自动记录错误日志 |
| C2.4 | 用户不能手动触发状态流转 |

### 验收标准

- [ ] 状态流转严格遵循状态机定义
- [ ] 每次流转都有时间记录
- [ ] 异常流转被系统拒绝并记录

## M3: 任务详情

### 职责
展示单个任务的完整信息，包括执行日志、错误信息、关联产物。

### 信息结构

```
任务详情
├── 基本信息：名称、agent、状态、创建时间
├── 时间线：各状态进入/退出时间
├── 执行日志：工具调用记录
├── 产物链接：PRD / 代码 / 测试报告
└── 错误信息：失败时展示完整堆栈
```

### 限制

| 编号 | 限制内容 |
|------|----------|
| C3.1 | 执行日志最多保留最近100条 |
| C3.2 | 错误堆栈最大显示2000字符，超长截断并提供下载 |
| C3.3 | 产物链接仅展示有效链接，无效时显示"未生成" |
| C3.4 | 不支持任务详情页修改任务状态 |
| C3.5 | 关闭详情页不触发任何状态变更 |

### 验收标准

- [ ] 任务详情完整展示所有字段
- [ ] 执行日志按时间倒序
- [ ] 失败任务显示完整错误信息
- [ ] 产物链接可点击跳转

## 跨模块约束

| 编号 | 约束内容 |
|------|----------|
| MX.1 | 任务看板数据由 agent 执行事件驱动，不支持手动创建任务 |
| MX.2 | 任务看板刷新周期 ≤ 5 秒 |
| MX.3 | 移动端（<768px）看板列水平滚动，不折叠 |
| MX.4 | 首页执行进度与任务看板数据同源，保持一致 |
| MX.5 | 产物交付模块的 PRD/代码链接与任务看板的产物链接同步 |
