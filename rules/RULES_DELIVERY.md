## M1: PRD 版本管理

### 职责
存储并展示所有经过评审确认的 PRD 版本，用户可查看任意历史版本详情。

### 版本定义

| 类型 | 触发条件 | 版本号 |
|------|----------|--------|
| **正式版** | 评审通过并确认 | v1.0, v2.0, v3.0 |
| **修订版** | 小薇妹审阅后修改 | v1.1, v1.2, v2.1 |

### 信息结构

每个 PRD 版本展示：
- 版本号（如 v1.0）
- 需求标题
- 评审状态（已通过 / 评审中 / 已作废）
- 完成时间
- 评审耗时
- 关联任务链接
- 操作：查看 / 回退（仅限已通过版本）

### 限制

| 编号 | 限制内容 |
|------|----------|
| C1.1 | 仅展示已通过评审的正式版本（已作废版本不可见） |
| C1.2 | PRD 正文存储为 Markdown，支持富文本渲染 |
| C1.3 | 版本列表按完成时间倒序 |
| C1.4 | 单次仅能回退到一个正式版本 |
| C1.5 | 回退后生成新的修订版（如 v1.1），原版本标记为"已作废" |
| C1.6 | 不支持删除已确认的 PRD 版本 |
| C1.7 | PRD 详情页展示完整版本历史（含所有修订版） |

### 验收标准

- [ ] 所有已通过评审的 PRD 版本正确展示
- [ ] 点击版本号展开完整 PRD 内容
- [ ] 版本历史清晰展示修订轨迹
- [ ] 已作废版本在详情页可查，但不在列表主视图展示

## M2: 代码交付

### 职责
关联展示各 PRD 版本对应的代码交付物，提供直跳 GitHub 的链接。

### 信息结构

每个 PRD 版本关联：
- 仓库地址（GitHub 个人仓库）
- 对应分支名
- 最新 Commit SHA（前7位）
- Commit 时间
- 代码行数统计（新增/删除）
- CI 状态（通过 / 失败 / 进行中）

### 限制

| 编号 | 限制内容 |
|------|----------|
| C2.1 | 仓库地址仅支持 GitHub 个人仓库 |
| C2.2 | 代码链接直接跳转 GitHub 仓库，不做内嵌展示 |
| C2.3 | 不支持代码下载（用户自行跳转 GitHub） |
| C2.4 | CI 状态刷新周期 ≤ 30 秒 |
| C2.5 | Fork ai_study 仓库禁止（配置层面拦截） |
| C2.6 | 代码行数统计基于 Git diff，不含依赖包 |
| C2.7 | PRD 与代码分支一一对应，命名规则：`prd-{版本号}`（如 `prd-v1.0`） |

### 验收标准

- [ ] PRD 版本正确关联对应代码分支
- [ ] GitHub 链接可正常跳转
- [ ] CI 状态实时更新
- [ ] 代码行数统计准确

## M3: 回退机制

### 职责
支持将项目回退到任意历史 PRD 版本，并同步触发对应代码分支的回退。

### 回退流程

```
用户发起回退（选择目标版本 v1.x）
        ↓
系统校验：该版本是否可以回退
        ↓
确认回退（显示影响范围：PRD + 代码）
        ↓
执行回退：
  1. PRD 状态置为"已作废"
  2. 生成新修订版（如 v1.x+1）
  3. 代码切换到对应分支
        ↓
通知相关 agent 重新评审
        ↓
完成后展示回退结果
```

### 回退规则

| 条件 | 是否允许回退 |
|------|-------------|
| 目标版本为已通过评审的正式版 | ✅ 允许 |
| 当前版本为最新正式版 | ❌ 不允许（无可回退目标） |
| 目标版本已被作废 | ❌ 不允许 |
| 存在未完成的进行中任务 | ⚠️ 警告确认后再执行 |

### 限制

| 编号 | 限制内容 |
|------|----------|
| C3.1 | 回退操作需要用户二次确认（弹窗展示影响范围） |
| C3.2 | 回退后自动生成修订版，原版标记为"已作废" |
| C3.3 | 代码同步回退到对应分支，不支持仅回退 PRD |
| C3.4 | 回退操作记录到执行日志 |
| C3.5 | 回退完成后通知 PM agent 重新评审 |
| C3.6 | 回退操作本身不可回退 |
| C3.7 | 回退耗时预计值显示，实际耗时记录 |

### 验收标准

- [ ] 可回退版本显示"回退"按钮
- [ ] 不可回退版本按钮禁用并显示原因
- [ ] 回退前正确展示影响范围
- [ ] 回退后 PRD 与代码状态一致
- [ ] 回退操作有完整记录
