## 1. 团队角色

| 角色 | 负责人 | 职责 |

| **总指挥** | 小毛球 | 协调分配任务，跟踪进度，向老板汇报 |

## 2. Sprint节奏

- **周期**：2周
- **启动日期**：2026-03-22（Sprint 1）

---

## 3. 项目流程

### 单模块流程

```
PRD输出 → 评审 → 开发工程师开发 → 测试工程师测试 → 产品经理验收 → 运维工程师维护
                ↓                              ↓
              老板终审（每模块完成后）
```

### 多模块并行调度

- 下一模块PRD可在当前模块开发时同步输出
- 测试工程师测完当前模块立即接续下一模块
- 流水线不停，最大化并行效率

### 模块验收节点

每模块完成后，必须提供以下两份文档作为老板终审的必要输入：
1. **测试报告**（由测试工程师输出）
2. **验收意见**（由产品经理输出）

---

## 4. 铁律（不可违背）

1. **需求达不到验收标准不开发**
2. **PRD不经评审不开发**
3. **PR不经1-2人review不合并**
4. **main分支不过CI门禁不合并**

---

## 5. 评审流程

```
PRD输出 → 开发+测试评审 → 总指挥复审 → 老板终审
```

- PRD评审参与方：开发工程师、测试工程师
- 复审：总指挥（小毛球）
- 终审：老板（小薇妹）

---

## 6. 汇报机制

- **每日汇报**：向老板汇报项目进度

---

## 7. 异常处理原则

- 发现问题先找根因，不盲目修复
- 回溯流程，检查是否有定义不清晰之处

---

## 8. CI/CD门禁

```
代码提交 → 编译构建 → 语法/风格检查 → 单元测试 → 安全扫描 → 集成测试 → 部署审批 → 发布上线
```

任一关卡失败，代码不得进入下一阶段。
