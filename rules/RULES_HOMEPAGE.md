# 首页模块规则 v1.4

## 模块结构

```
HomePage
├── M1: 需求提交入口
└── M2: 实时动态流
```

---

## M1: 需求提交入口

### 职责
为用户提供低门槛的需求提交通道，接收并初步校验用户输入后转化为系统任务。

### 输入
用户填写表单或上传文档，两种方式二选一：

**方式一：+ 新增需求（自然语言）**
| 字段 | 类型 | 约束 |
|------|------|------|
| 需求标题 | 字符串 | 必填，≤50字符 |
| 需求描述 | 字符串 | 必填，≤500字符 |
| 优先级 | 枚举 | 选填，MoSCoW：Must / Should / Could / Won't |
| 预期上线 | 字符串 | 选填，版本号格式如 v1.0 |

**方式二：+ 上传需求文档**
| 字段 | 类型 | 约束 |
|------|------|------|
| 文档标题 | 字符串 | 必填，≤50字符，来源为文件名（可编辑） |
| 文档附件 | 文件 | 必填，支持 .md / .docx / .txt / .pdf，单文件≤10MB |
| 优先级 | 枚举 | 选填，MoSCoW：Must / Should / Could / Won't |
| 预期上线 | 字符串 | 选填，版本号格式如 v1.0 |

**方式三：+ 优化建议（自然语言）**
| 字段 | 类型 | 约束 |
|------|------|------|
| 建议标题 | 字符串 | 必填，≤50字符 |
| 优化目标 | 字符串 | 必填，关联至现有PRD编号或功能点 |
| 优化内容 | 字符串 | 必填，≤500字符 |

> **说明：** 自然语言场景下用户可能只写一句话，故字数上限设为500字符；文档上传场景无字数限制。

### 输出
- 提交成功：
  - 自然语言 → 生成 PRD 草稿，状态置为"评审中"，触发 PM agent 评审
  - 文档上传 → 解析文档内容 → 生成 PRD 草稿，状态置为"评审中"，触发 PM agent 评审
- 提交失败：返回错误信息，保留用户已填写内容

### 限制
| 编号 | 限制内容 |
|------|----------|
| C1.1 | 自然语言字数限制为硬性上限，超限禁止提交并提示用户 |
| C1.2 | 文档格式仅支持 .md / .docx / .txt / .pdf，不支持其他格式 |
| C1.3 | 单文件大小上限 10MB，超限禁止上传并提示用户 |
| C1.4 | 提交后最长处理时间为 60 秒（包含文档解析时间） |
| C1.5 | 提交动作防抖：2秒内连续点击仅触发一次提交 |
| C1.6 | 网络错误时保留用户输入，不清空表单 |
| C1.7 | 提交成功后关闭抽屉并展示结果提示 |
| C1.8 | 抽屉关闭时若表单有内容，弹出二次确认 |
| C1.9 | 不支持草稿保存，用户未提交内容不持久化 |
| C1.10 | 文档上传与自然语言描述互斥，一次提交仅能选其一 |

### 验收标准
- [ ] 字数超标时输入框红色边框 + 实时提示，不允许提交
- [ ] 文件格式不符时上传区红色边框 + 提示，不接受文件
- [ ] 文件大小超限时上传区提示，不接受文件
- [ ] 提交中按钮置灰，2秒防抖生效
- [ ] 网络错误后点击重试，用户输入完整保留
- [ ] 提交成功后抽屉正确关闭并显示成功提示
- [ ] 关闭抽屉时未保存内容正确触发确认弹窗

## M2: 实时动态流

### 职责
按时间倒序记录并展示所有 agent 的工具调用行为，供用户追溯系统执行历史。

### 输入
每次 agent 工具调用事件，事件结构：
```
{
  "时间戳": ISO8601,
  "执行者": agent名称,
  "动作": "调用工具: xxx",
  "关联任务": 任务名称,
  "状态": "成功 | 失败",
  "错误信息": "失败时填写"
}
```

### 输出
动态记录列表，每条记录显示：时间 / 执行者 / 动作 / 关联任务 / 状态

### 限制
| 编号 | 限制内容 |
|------|----------|
| C2.1 | 最多保留最近50条记录，超出后自动丢弃最旧记录 |
| C2.2 | 刷新周期固定为10秒，不支持自定义 |
| C2.3 | 失败记录视觉标记为红色，点击可展开错误详情 |
| C2.4 | 不支持用户手动删除或编辑动态记录 |
| C2.5 | 列表按时间倒序排列，不支持排序方式切换 |

### 验收标准
- [ ] 工具调用事件在发生后10秒内出现在动态流
- [ ] 失败记录有明显视觉区分
- [ ] 记录数达到50条后，新记录正确插入，旧记录正确淘汰
- [ ] 点击失败记录可展开完整错误信息

## 跨模块约束

| 编号 | 约束内容 |
|------|----------|
| MX.1 | 各模块数据独立更新，不因某一模块失败影响其他模块展示 |
| MX.2 | 首页不支持自定义布局模块顺序 |
| MX.3 | 移动端（<768px）两模块垂直堆叠，动态流在上，需求入口在下 |
